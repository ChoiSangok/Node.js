+ 미들웨어

 - 익스프레스의 핵심 
 - 요청과 응답의 중간에 위치 
 - 요청과 응답을 조작하여 기능을 추가하기도 하고, 나쁜요청을 걸러냅니다.
 - next 함수를 이용해서 다음 미들웨어로 현재 요청을 넘길 수 있음.
 - app.use 안에 있는 함수들은 모두 미들웨어이며, 요청이 올때마다 미들웨어를 거치며 클라이언트에게 응답
 
 
+ 특징
 - 모든 코드를 실행
 - 다음 미들웨어 호출
 - res, req 객체 변경 가능
 - 요청 - 응답 주기를 종료
 
+ 종류
 1. 어플리케이션 : 전체 영역에서 처리 가능, 앱에 대한 request가 발생할 때마다 실행되는 미들웨어
    app.use()  app.METHOD()
 2. 라우터 : 라우더 단위로 request가 발생하면 실행, 특정 지정 라우터가 실행되었을 때마다 실행외는 미들웨어
 
 3. 에러 핸들링 : 에러가 발생하면 핸들링 할 때 사용하는 미들웨어

 + Express 미들웨어
